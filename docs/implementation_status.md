## Phase 1 (A Tree and an Elf)
Complete. Ten species implemented: Elf, Capybara, Boar, Deer, Elephant, Goblin, Monkey, Orc, Squirrel, Troll (all with procedural sprites, data-driven behavior). Goblin/Orc/Troll are hostile-faction placeholders — spawnable via debug UI, wander and climb, no food decay or mood system yet. Dynamic pursuit infrastructure: tasks can track a moving target creature with automatic repathfinding (prerequisite for combat).

## Phase 3 (Combat)
Projectile system complete: ballistics math module (integer-only sub-voxel coordinates, symplectic Euler trajectory stepping, iterative aim solver), projectile entity table in SimDb (inventory-based payload, FK nullify on shooter), ProjectileTick batched event, collision detection (solid voxels + creature spatial index), momentum-based damage (linear in impact speed), impact resolution (surface → ground pile, creature → damage + ground pile, OOB → despawn), DebugSpawnProjectile command, rendering (oriented cylinder meshes via projectile_renderer.gd). Creature spatial index complete (BTreeMap voxel→creatures, maintained at all position mutation points, multi-voxel footprint support). HP/death system complete with incapacitation: per-species hp_max, VitalStatus (Alive/Incapacitated/Dead), DamageCreature/HealCreature/DebugKillCreature commands. At 0 HP creatures become Incapacitated (abort action, emit event, notification) instead of dying instantly; bleed-out at 1 HP per heartbeat, true death at -hp_max. Massive hits past -hp_max kill outright. Healing above 0 HP revives to Alive. Incapacitated creatures remain in spatial index and are targetable but cannot act. Death handler (task interruption, inventory drop, spatial index deregistration, notification). Dead creatures remain in DB (vital_status=Dead) for future states (ghost, undead). HP bars in UI: overhead sprite bars (visible only when damaged, green/yellow/red; dark gray on black for incapacitated), incapacitated sprites rotated 90° sideways, creature info panel shows "INCAPACITATED" status label. Melee strike action complete: per-species melee_damage/melee_interval_ticks/melee_range_sq config, ActionKind::MeleeStrike with cooldown, closest-point AABB range check (multi-voxel footprint support), DebugMeleeAttack command, CreatureDamaged event. Ranged shoot action complete: ActionKind::Shoot with configurable cooldown (shoot_cooldown_ticks), requires bow + arrow in inventory, line-of-sight check (VoxelWorld::has_los DDA ray march, leaf/fruit transparent, multi-voxel footprint support), aim feasibility via existing iterative solver (no artificial range cap), DebugShootAction command, ProjectileLaunched event, arrow consumed from inventory on fire. Friendly-fire avoidance: pre-shot ballistic trajectory check skips non-hostile creatures near origin (squad can stand together), blocks shots through friendlies in the flight path; in-flight projectiles also skip friendly creatures near origin. When blocked: alternate target selection (try other hostiles with clear paths), ranged repositioning to neighboring nav nodes scored by clear-shot availability, avoidance of blocking other archers' lines of fire, and fallback non-blocking repositioning. Hostile AI with data-driven detection: unified `EngagementStyle` struct on both `SpeciesData` and `MilitaryGroup` with weapon preference (PreferRanged/PreferMelee), ammo exhaustion behavior (SwitchToMelee/Flee), engagement initiative (Aggressive/Defensive/Passive), and disengage threshold (HP%). Per-species `hostile_detection_range_sq` (squared 3D euclidean voxels). `detect_hostile_targets()` scans spatial index with distance filter and faction-aware hostility rules. Aggressive creatures pursue nearest detected target via Dijkstra; defensive creatures only pursue within `defensive_pursuit_range_sq`. Auto-melee when adjacent, weapon-preference-aware ranged/melee selection. Task preemption priority system complete. Flee behavior: creatures with passive initiative or HP below disengage threshold detect hostiles within `hostile_detection_range_sq` and perform greedy retreat (maximize distance from nearest threat per activation step), interrupting current tasks. Player-directed attack task (AttackCreature command): creates AttackTarget task with PlayerCombat preemption, pursues target with melee/ranged combat, completes on target death, configurable path retry limit. Player-directed goto (DirectedGoTo command): assigns GoTo task directly to a specific creature with preemption. Attack-move command (F + click): walk to destination while engaging hostiles en route, nearest-target selection, disengage-on-path-failure. Right-click context commands in selection_controller.gd: attack hostile, move-to friendly/ground. RTS selection: stable CreatureId addressing (replaced fragile species+index), box selection with click-drag rectangle, multi-creature selection with Shift modifier, group info panel (`group_info_panel.gd`) showing sprites/names/activity for multi-select, faction-colored selection highlight rings at creatures' feet (`selection_highlight.gd`). Military groups: `MilitaryGroup` table (auto-increment PK, civ FK cascade), `EngagementStyle` per-group (replaces old Fight/Flee toggle), implicit civilian membership (`creature.military_group = None`), worldgen default groups (Civilians + Soldiers), 5 SimAction commands (create/delete/reassign/rename/set engagement style), group-aware `should_flee`/`wander` behavior, Military [M] toolbar button and panel UI (`military_panel.gd`) with summary/detail pages, engagement style controls (initiative/weapon/ammo cycle buttons + disengage threshold slider), rename, delete, member reassignment. Military equipment acquisition: `MilitaryGroup.equipment_wants` (Vec of item kind + material filter + quantity), default Soldiers group starts with 1 bow + 20 arrows, heartbeat Phase 2b¾ drops unowned items not matching group wants and creates `AcquireMilitaryEquipment` tasks (same flow as `AcquireItem` but no ownership transfer), `SetGroupEquipmentWants` command, reusable `WantsEditor` widget shared by building logistics and military equipment UIs; wearable military equipment auto-equips on pickup (displacing existing items in the same slot), duplicate equip-slot validation rejects conflicting wants with a notification, picker UI hosted in a middle-column panel. Voxel exclusion: creatures cannot move into voxels occupied by living hostile creatures. Enforced at move time via `destination_blocked_by_hostile()` (checks all destination footprint voxels against spatial index). Handles multi-voxel footprints (2x2x2). Applied in `move_one_step()`, `random_wander()`, `flee_step()` (with fallback if cornered), and `walk_toward_*()` inline movers. Blocked creatures retry after `config.voxel_exclusion_retry_ticks`. Item durability system: `ItemStack` has `current_hp`/`max_hp` fields, per-`ItemKind` max HP in config, `inv_damage_item()` with stack splitting and `ItemBroken` events. Condition labels in display names: "(worn)" at ≤70% HP, "(damaged)" at ≤40% HP (configurable thresholds). Arrow durability on impact: random damage in configurable `[min, max]` range (default 0–3), destroyed arrows removed, survivors placed in ground piles. Worn/damaged arrows deal proportionally less creature damage (scaled by current_hp/max_hp, minimum 1). GDScript `ItemUtils.condition_label()` mirrors Rust logic. Armor damage reduction: equipped armor subtracts from incoming damage (melee + projectile), minimum 1 damage always gets through. `effective_armor_value()` with `ArmorParams` applies condition-based penalties (worn: -1, damaged: -2, with per-piece floors). Armor/clothing degradation on combat hit: weighted random body location pick, penetrating hits roll `[0, 2*penetrating_damage]` durability loss, non-penetrating hits have 1/20 chance of 1 HP loss. All parameters configurable in `GameConfig`. Debug-triggered enemy raids: `TriggerRaid` command picks a random hostile civ, spawns `raid_size` creatures at the forest floor perimeter (per-species: Goblin 6, Orc 3, Troll 2), attack-moves each toward the tree. `CivSpecies::to_species()` conversion, `spawn_creature_with_civ()` for explicit civ affiliation. Melee weapons: Spear (extended reach, range_sq=8) and Club (high damage, range_sq=3) item types with GrowSpear/GrowClub workshop recipes. Weapon selection picks best weapon by distance (highest damage whose range covers target, bare-hands fallback). Weapon damage replaces species base; STR scaling still applies. Melee weapon degradation on strike (configurable 0–2 HP per hit). Troll HP regeneration: species-level `ticks_per_hp_regen` config (0 = disabled), batch-applied at heartbeat via integer division, clamped to `hp_max`; trolls default to 500 (2 HP/sec). Regen continues during incapacitation, racing bleed-out — trolls recover if not finished off (net +5 HP/heartbeat). Creature gravity (F-creature-gravity): unsupported creatures fall to nearest valid surface with configurable damage per voxel (default 10 HP/voxel). Support rules vary by species type: flying creatures exempt, climbers need nav node, ground-only creatures also need solid below, 2x2x2 creatures checked via large nav graph. Detected at activation time and via LogisticsHeartbeat sweep. DeathCause::Falling, CreatureFell event, task/path/action cleanup on fall. Degenerate fallback teleports to nearest nav node.

## Phase 4 (Economy and Ecology)
Kitchen cooking, workshop manufacturing (bow/arrow/bowstring recipes) with Grow-verb mana drain (multi-action progress, wasted-action abandon), elf personal item acquisition, creature thoughts, and basic mood scoring implemented. Notification system with sim-side persistence (SimDb table), multiplayer-aware command pipeline, toast UI, moping notifications, notification bell icon (procedural gold bell with red unread badge), and notification history panel (full-height right-side scrollable log, newest-first, hydrates from DB on save load, 500-entry cap with oldest eviction). Item quality system done (F-item-quality): coarse-grained quality tiers (Crude/Fine/Superior) rolled at craft time via quasi_normal + crafter stats + skill against thresholds; input quality drag-down propagation; quality prefix in display names; starting gear Crude for early progression. Creature actions formalized as typed, duration-bearing operations (`ActionKind` enum, `MoveAction` table for interpolation). Unified `interrupt_task()` entry point for all task interruption and cleanup (nav invalidation, mope preemption, pursuit abandonment). Logistics material filtering: `MaterialFilter` enum (`Any`/`Specific(Material)`) threads through all inventory and logistics functions, `hauled_material` tracking on haul tasks for precise in-transit counting, dynamic two-step UI picker (item kind → material filter). Proximity-based task assignment: idle creatures pick the nearest available task by Dijkstra nav-graph distance; rest not started. Wild grazing done (F-wild-grazing): herbivore food cycle with grass search, graze task resolution, grassless voxel tracking, timed grass regrowth, and grassless mesh rendering.

## Phase 5 (Quality of Life)
Partial — double-click group select (F-dblclick-select): double-clicking a player-civ creature selects all on-screen creatures in the same military group; civilians treated as implicit group. SC2-style selection groups (F-selection-groups): Ctrl+1–9 saves current selection, Shift+1–9 adds to group, 1–9 recalls, double-tap centers camera on group centroid. Groups persist across save/load via SimDb `selection_groups` table (per-player). GDScript keeps authoritative local copy for instant recall. Command queue (F-command-queue): shift+right-click appends commands (GoTo, AttackMove, AttackCreature) to a creature's sequential queue via `restrict_to_creature_id` and `prerequisite_task_id` fields on the Task table. Unshifted commands replace the queue. Queue survives autonomous interruptions (flee, hunger, sleep) — only player commands and natural completion clear it. `find_queue_tail` discovers orphaned queue chains when `current_task` is None (e.g., mid-flee). Cycle guard caps chain traversal at 256 iterations.

## Creature Biology
In progress — biological traits stored as sim data (`creature_traits` table with compound unique index on creature_id + trait_kind). `TraitValue` sum type (Int/Text) covers palette indices, seeds, and future stats. 18 `TraitKind` variants across all 10 species. Traits rolled at spawn via PRNG → Knuth hash derivation (1 PRNG call per creature). Sprite generation reads trait indices directly instead of re-deriving from seed hash — `CreatureDrawInfo` carries trait indices, `params_from_traits` on all species. `Role` enum dropped from elf sprites (equipment overlays handle clothing). `trait_int`/`trait_text` helpers with fallback defaults. `BioSeed` trait stored per creature for future trait derivation. Creature stats done (8 stats: STR/AGI/DEX/CON/WIL/INT/PER/CHA with exponential 2^(s/100) scaling, wired to melee damage, move speed, HP, arrow deviation, mana). Creature skills in progress — 17 skill `TraitKind` variants (Striking, Archery, Evasion, Ranging, Herbalism, Beastcraft, Cuisine, Tailoring, Woodcraft, Alchemy, Singing, Channeling, Literature, Art, Influence, Culture, Counsel) with info panel Skills tab, probabilistic advancement with INT scaling wired to melee/ranged combat, construction (woodsinging), crafting (per-verb), and harvesting; speed effect done (additive stat+skill reduces task duration via apply_stat_divisor), quality effect done (F-item-quality: craft-time quality rolls use verb-specific stat+skill pairings). ## Tabulosity (sim DB)
Typed in-memory relational store complete — derive macros for `Bounded`, `Table`, `Database` with FK validation and serde support (feature-gated). Includes compound (multi-column) primary keys (`#[primary_key("f1", "f2")]`) with tuple key types, compound indexes (`#[index(...)]`) with prefix queries, filtered/partial indexes, unified `IntoQuery` API, tracked runtime bounds, `on_delete cascade`/`nullify` FK semantics with cycle detection, auto-increment primary keys (`#[primary_key(auto_increment)]`), parent-PK FKs (`pk` keyword for 1:1 relations), unique index enforcement (`#[indexed(unique)]`), `modify_unchecked` closure-based in-place mutation with debug-build safety checks, `QueryOpts` for ordering (asc/desc) and offset (skip N) on all query methods, `modify_each_by_*` query-driven batch mutation, and schema versioning (`#[schema_version(N)]`) with missing-tables-default-to-empty on deserialization. **Integrated into `elven_canopy_sim`:** `SimDb` (45 tables) replaces all BTreeMap entity storage — players, creatures, tasks (with decomposed extension tables), blueprints, structures, inventories, item stacks (with subcomponents and enchantments), ground piles, thoughts, notifications, furniture, music compositions, logistics wants, civilizations, civ relationships, military groups, trees, and great tree info.
